# 大觀國中畢業成年禮闖關網頁使用說明

## 目前完成內容

這是一個靜態網頁版的 Supabase 闖關系統，包含：

- 學生登入：901 到 914 班、第1組到第10組、組員人數、座號。
- 老師組登入：全校共用老師組，使用教師私下設定的密碼。
- 兩大主題：硬漢嶺挑戰、淡水玩很大。
- 拍照上傳與答案儲存。
- 已送出的照片與答案允許修改。
- 自動判讀文字答案。
- 老師後台查看照片、答案、審核通過或退回補件。
- 老師後台可將指定班級或全部學生「已送出」的項目批次通過。
- 老師後台可刪除單組、指定班級或全部學生資料，刪除前會跳出確認。
- 完成速度排行榜。
- Supabase Database 與 Supabase Storage 儲存。
- 沒有 Supabase 設定時，可用示範模式先測流程。

## 檔案說明

- `index.html`：主網頁。
- `styles.css`：畫面樣式。
- `app.js`：登入、闖關、上傳、後台、排行榜邏輯。
- `supabase-schema.sql`：Supabase 建表、照片 bucket、權限規則。
- `畢業成年禮闖關網頁_設計藍圖.md`：需求與設計藍圖。

## 本機測試

直接打開 `index.html` 就可以測試。

若尚未設定 Supabase，網頁會顯示「示範模式」。示範模式資料只存在目前這台電腦的瀏覽器，不會同步到其他手機或電腦。

## Supabase 設定步驟

1. 到 Supabase 建立一個新專案。
2. 開啟 Supabase 專案中的 SQL Editor。
3. 貼上 `supabase-schema.sql` 的全部內容並執行。
4. 新版 `supabase-schema.sql` 會在尚未設定密碼時，預設建立老師密碼 `8888`。如果要改成其他密碼，再另外執行下面這行：

```sql
select public.set_teacher_password('請改成老師私下知道的密碼');
```

如果曾經出現 `function gen_salt(unknown) does not exist`，請重新執行最新版 `supabase-schema.sql`。新版已改用 PostgreSQL 內建 `md5()`，不再依賴 `gen_salt()`。

5. 到 Project Settings 找到：
   - Project URL
   - anon public key
6. 第一次設定網頁時，用網址加上 `?setup=1` 開啟設定頁，例如 `index.html?setup=1`。
7. 貼上 Project URL 與 anon public key。
8. 儲存後，若連線成功，首頁會顯示「目前使用 Supabase 正式儲存」。

重要提醒：不要把 service role key 貼到網頁中，只能使用 anon public key。

## 學生操作方式

1. 點「開始闖關」。
2. 選擇班級、組別、組員人數。
3. 輸入組員座號。
4. 進入任務總覽。
5. 依序進入關卡，上傳照片或回答問題。
6. 若送出後發現照片不清楚或答案要修改，可重新進入同一題更新。
7. 完成全部必要任務後，會進入完成速度排行榜。

## 老師操作方式

1. 點「開始闖關」。
2. 小組選擇「老師組」。
3. 輸入教師私下設定的密碼。
4. 登入後點「後台」。
5. 可依班級與審核狀態篩選。
6. 查看學生照片與答案。
7. 對每一題按「通過」或「退回補件」。
8. 若需要快速處理，可在「批次通過審核」按指定班級或全部學生批次通過。
9. 點「排行榜」可查看完成速度。

## 批次通過審核

老師登入後進入「後台」，可以批次通過學生已送出的項目：

- 一整班通過：先在班級篩選選定班級，再按「通過某班已送出項目」。
- 全部通過：按「通過全部已送出項目」。

每次批次通過前都會跳出「是否確認」。按確認後，系統只會把已經送出的答案與照片標記為通過；尚未送出的任務不會被系統自動新增。

## 刪除測試資料

老師登入後進入「後台」，可以刪除測試資料：

- 單組刪除：在該小組右側快速狀態按「刪除此組」。
- 一整班刪除：先在班級篩選選定班級，再按「刪除某班資料」。
- 全部刪除：按「刪除全部學生資料」。

每次刪除前都會跳出「是否確認刪除」。按確認後才會刪除。正式 Supabase 版會先透過 Supabase Storage API 刪除照片，再刪除小組資料、答案、照片紀錄、審核紀錄與排行榜資料。

若後台出現刪除函式不存在或刪除失敗，請重新執行最新版 `supabase-schema.sql`。

若曾經出現 `Direct deletion from storage tables is not allowed`，請更新刪除函式為最新版。原因是 Supabase 不允許 SQL 直接刪 Storage 內部資料表，必須透過 Storage API。

## 自動判讀規則

系統會自動判讀文字題：

- 完全符合標準答案：顯示自動判讀正確。
- 符合關鍵字：顯示接近正確。
- 不符合：顯示待修正。
- 照片題與自由回答題：不直接判錯，由老師後台審核。

硬漢嶺牌樓題會提醒學生照片中必須清楚出現：

```text
挑路要挑難路走、挑擔要挑重擔挑
```

英國領事館餐桌題接受：

```text
呼叫僕人
叫僕人
呼叫僕役
叫服務人員
呼叫服務人員
```

## 照片保存

照片上傳到 Supabase Storage 的 `challenge-photos` bucket。

系統會記錄照片預計刪除時間，設定為上傳後 30 天。活動結束一個月後，老師可到 Supabase Storage 批次刪除照片。文字答案、審核紀錄與排行榜資料可另外保留。

## 部署建議

正式使用時可把這幾個檔案放到：

- GitHub Pages
- Netlify
- Vercel
- 學校自己的網站空間

網頁部署平台只負責放網頁，照片與資料仍存 Supabase。

## 安全提醒

目前版本已把教師密碼從首頁與前端程式移除。正式 Supabase 版會呼叫資料庫函式驗證密碼，學生不會在首頁看到密碼。

因為這是靜態網頁，真正高安全性的後台需要再加入 Supabase Auth、教師帳號或 Edge Functions。若活動照片不希望任何知道網址的人都能透過技術方式讀取，正式上線前建議改成教師帳號登入版。

## 圖片沒有顯示時

本版已把 Google Drive 圖片網址改成較適合網頁嵌入的 thumbnail 格式。

若圖片仍然沒有顯示，請到 Google Drive 檢查每一張圖片的分享權限：

```text
共用 > 一般存取權 > 知道連結的任何人 > 檢視者
```

如果權限仍維持「限制」，學生手機就看不到圖片。若學校網路會擋 Google Drive 圖片，建議把活動圖片改存到 Supabase Storage 或網站同一個資料夾中。
